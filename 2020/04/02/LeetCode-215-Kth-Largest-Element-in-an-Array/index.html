215. Kth Largest Element in an Array

Medium

Description

Go to LeetCode

Solution 1

最直接的方法就是排序，然后直接选择第 K 大的元素即可。

时空复杂度完全取决于排序算法的复杂度，就常用的快速排序而言，期望意义下的时间复杂度为 $O(n\log n)$ 。

Solution 2

利用快速排序的思想。

快速排序中每次 partition 都会确定一个枢轴元素 pivot 的最终位置，并且枢轴左边的元素都比它小，右边的元素都比它大（或者反过来，取决于升序还是降序）。

根据这个性质，一次 partition 过后如果 pivot 恰好是第 K 个元素，那么已经找到了目标，算法结束；否则可根据 K 与 pivot 的位置的大小关系在 pivot 的左边或者右边寻找目标元素，这样每次 partition 都排除了一部分元素，能够排除多少取决于选择的 pivot 的质量。

Remark

对于选择第 K 大元素的问题，可用的方法非常多，处了上述提到的两种，还可以用堆排序方法。此外，C++ STL 中内置了 nth_element 和 partial_sort 函数，可以直接调用来解决此问题。最后，实际上有所谓 线性时间选择算法 ，其最坏情形下时间复杂度仍然为 $O(n)$ 。当然，该算法只是理论意义比较重大，在实际应用中还是上面的 快速选择算法 综合性能最好，它和快速排序实际上是同一个人提出的，原理和特性几乎也是相同的。
