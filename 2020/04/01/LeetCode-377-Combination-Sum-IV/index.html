相对于 原问题 Combination Sum 有两点变化：

  • 结果集的不同的序列视为不同的 combination ，换句话说，此问题实质上是一个 permutation sum 问题而非 combination sum 。

仔细分析方法 1 中的代码可以发现回溯过程中可能包含了一些重复计算，因为一旦我们在某次回溯中计算出了某个特定的 target 对应的结果，在后续的回溯中如果再次遇到这个 target 会重复对其进行递归计算。故可以用 DP 的思想将已经计算出的结果存起来，每次计算前先查表看看是否已经有结果，这种自顶向下的递归式 DP 实现，或称记忆化搜索的代码如下：

对于该状态转移方程，有必要研究一下 dp 数组的初始状态和初始值，因为该方程的初始化不向其他 DP 问题一样直观。从上述方程可以看出，只有当 nums[j] <= i 时，等号的右边才存在有效项，才能够根据这一方程更新当前的 dp[i] 。因此，记 m 为 nums 数组中的最小值，那么所有的 dp[i], i<m 都是初值，它们不能通过上述方程进行更新，而且会对后面的值产生影响，因此必须初始化。根据 dp 数组的定义，如果 target < m 则不存在符合题意的 combination ，所以应该把所有的 dp[i], i<m 都初始化为 0 。

Follow up

对于 follow up，如果允许 nums 中含有负数，可能会导致该问题有无穷多解。因为每个数都可以使用任意多次，故和为 0 的一组数可以任意多次地添加到 combination 中，而不改变其和。

这种情形下可添加限制 nums 中的每个数最多只能使用一次。
